# Git Commands Guide

A beginner-friendly guide to essential Git commands, workflows, and pull request processes for collaborative projects.

---

## About

This repository serves as a practical reference for developers who are new to Git and GitHub. It walks through everything from initial setup and configuration to day-to-day commands, branching strategies, and the pull request review cycle. Whether you're making your first commit or trying to recover from a bad merge, this guide has you covered.

The primary resource is [**GIT_GUIDE.md**](GIT_GUIDE.md), which contains detailed explanations, example commands, and real-world workflows.

---

## What's Covered

- **Initial Setup** — Configuring your Git identity with `git config`
- **Initializing a New Repository** — Setting up a local project with `git init` and pushing it to GitHub for the first time
- **Core Commands** — `status`, `fetch`, `pull`, `add`, `restore`, `rm`, `commit`, `push`, `checkout`, `branch`, `log`, `reflog`, and `reset`
- **Common Workflows** — Switching branches, updating your local repo, staging and committing changes, pushing work, and undoing mistakes
- **Pull Request Workflow** — A step-by-step walkthrough of creating, submitting, and iterating on pull requests through GitHub
- **Branch Protection and Repository Access** — Visibility settings, collaborator roles, branch protection rules, and recommended configurations for solo and collaborative repos
- **Quick Reference Table** — A concise cheat sheet of every command and its purpose

Each command includes usage examples, expected output, and clear warnings for destructive operations so you know exactly what's safe and what isn't.

## How to Use This Guide

1. **Clone the repository:**
   ```bash
   git clone https://github.com/your-username/git-commands-guide.git
   cd git-commands-guide
   ```

2. **Open [GIT_GUIDE.md](GIT_GUIDE.md)** — this is the main guide containing all commands, workflows, and examples.

3. **Follow along** — try the commands in your own terminal as you read through each section.
